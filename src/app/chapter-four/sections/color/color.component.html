Colors by RGB and RGBa

You can create a color by combining different levels of red, green, and blue. This is called a RGB color. There are few limits on the colors that can be produced using this method. There are four ways to affect color in RGB.

Functional RGB Colors

You can use rgb(color) and color would use either a percentage (0-100%) or integer (0-255). Using the percent notation you could specify white and black:
  • rgb(100%,100%,100%) - font white
  • rgb(0%,0%,0%) - font black
Or using the integer-triplet notation the same colors would be:
  • rgb(255,255,255) - font white
  • rgb(0,0,0) - font black
Remember that you cannot mix integers and percentages in the same color value. YOu would change the rgb values to get different shades of colors.

RGBa colors

This is similar to RGB, except the a stands for alpha. This is a measure of opacity. You can do half transparent white and by:
  • rgb(100%,100%,100%,0.5)
  • rgb(255,255,255,0.5)
Completely transparent would have a value of 0 in the alpha position. rgb(0,0,0) and rgb(0,0,0,1) would be the same black color. Alpha are real numbers ranging from 0 to 1. You cannot us a percent as the alpha value.

Hexadecimal RGB colors

You can also use hexadecimal color notation to define a color. Such as color: #FF0000; which will set your color to red.
Hexadecimal notation is created by stringing three hexadecimal numbers together, the numbers range from 00 to FF. The generic syntax is #RRGGBB.
Hexadecimal notation is mathematically quivalent to the integer-pair notation discussed above. So, rgb(255,255,255) is the same as #FFFFFF.
You are able to only us three numbers, #666. This would be interpreted on the browser as #666666. YOu cannot short hand the hexadecimal value if the numbers are not all the same. You cannot use shorthand for #808080. If you use #888 that would be translated to #888888.

Hexadecimal RGBa colors

This adds a fourth number which is the alpha channel value. For black you would write #000000FF. This would be black with 100% opacity. These can also be shortened to be #000F.

Colors by HSL and HSLa

HSL stands for Hue, Saturation, and Lightness. Hue is for the hue angle in the range 0 - 360. Saturation is a percentage avlue from 0 (no stauration) to 100 (full saturation). Lightness is a percantage value from 0 (complete dark) to 100 (complete light).
Hue angle is used to show the full spectrum of colors with 0 degrees being red and proceeds through the colors of the rainbow ending with red again at 360.
Saturation measures the intensity of the color along the hue angle. Gray will always be the color when saturation is 0. Lightness tells how light or dark the color along the angle appears. A lightness of 0% will always be black, while 100% will always be white. Here are some examples:

p.one {{'{'}}color: hsl(0,50%,0%);{{'}'}}

p.two{{'{'}}color: hsl(60,50%,25%);{{'}'}}

p.three {{'{'}}color: hsl(120,50%,50%);{{'}'}}

p.four {{'{'}}color: hsl(180,50%,75%);{{'}'}}

p.five {{'{'}}color: hsl(240,50%,0%);{{'}'}}

p.six {{'{'}}color: hsl(300,50%,25%);{{'}'}}

p.seven {{'{'}}color: hsl(360,50%,50%);{{'}'}}

HSL also has an alpha value that can be added. The syntax is hsla(0,0%,0%,1);

Color Keywords

Two keywords can be used whereever colors are accepted. These are transparent and currentColor.
Transparent defines a completely transparent color. This is defined as rgb(0,0,0,0). This is more used for element background colors.
CurrentColor means the computed value of color for this element. For example: main {{'{'}} color: gray; border-color: currentColor; {{'}'}} this would give the border-color a color of gray or rgb(50%, 50%, 50%) because the is the color element.
